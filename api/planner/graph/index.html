planner.graph.sample_related_sub_goal_states(params: dexterity.planner.planner.parameter_container.ParameterContainer, goal_states: torch.FloatTensor, start_states: torch.FloatTensor) → torch.FloatTensor#

Generates related goal states based on the provided parameters.

Parameters:

  • params (ParameterContainer) – The container holding parameters of sub goal bounds.

  • goal_states (FloatTensor) – [num_parallel_searches, state_dimension] The goal states to generate related goal states from.

  • start_states (FloatTensor) – [num_parallel_searches, state_dimension] The start states to generate related goal states from.

Returns:

new_goal_states – [num_parallel_searches, state_dimension] Related goal states as a tensor.

Return type:

torch.FloatTensor

Note

This function assumes a diagonal covariance matrix. It relies on the fact that the entries are independent and identically distributed (i.i.d.) entries.

planner.graph.sample_feasible_states(plant: dexterity.planner.dynamics.mujoco_dynamics.MujocoPlant, bound_lower: torch.FloatTensor, bound_upper: torch.FloatTensor, size: int = 1, num_attempts: int = 5) → torch.FloatTensor#

Samples feasible states within given bounds, ensuring they are collision-free.

Parameters:

  • plant (MujocoPlant) – The simulation plant used to check and normalize states.

  • bound_lower (FloatTensor) – [state_dimension] Lower bounds for sampling states.

  • bound_upper (FloatTensor) – [state_dimension] Upper bounds for sampling states.

  • size (int) – The number of states to sample. Default is 1.

  • num_attempts (int) – The number of attempts to sample collision-free states. Default is 5.

Returns:

feasible_states – [size, state_dimension] Feasible states as a tensor, collision-free if possible.

Return type:

torch.FloatTensor

Note

If ignore_sampled_state_collisions is set in plant.params, the function will return normalized states directly. If no collision-free states are found after num_attempts, it will return the last set of normalized states sampled.
